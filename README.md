# Антитетрис

Антитетрис — это тетрис наоборот. Игрок не управляет падением фигуры напрямую: он выбирает, какую фигуру отдать компьютеру, а игра сама ищет для нее лучший поворот и лучшую позицию в стакане.


[video](./promo.mp4)

## Что это за игра

Вместо классического "собери линии сам" здесь другой конфликт:

- игрок принимает стратегическое решение;
- компьютер обязан разруливать последствия;
- каждая партия строится вокруг слабых мест алгоритма оценки поля.

Из-за этого игра проверяет не реакцию, а понимание формы: где появятся пустоты, какие рельефы опасны, когда выгодно ломать плоскость, а когда наоборот помогать алгоритму.

## Режимы

### `Versus`

Игрок саботирует компьютер.

- стартовый счет: `50`;
- на каждом ходу предлагаются `4` фигуры;
- на выбор есть `4` секунды;
- если время вышло, набор фигур обновляется, а игрок теряет `5` очков;
- если компьютер закрывает линию, игрок теряет еще `5` очков за каждую линию;
- если счет падает до `0`, побеждает компьютер;
- если фигуру больше некуда поставить, а очки еще есть, побеждает игрок.

Практически выгодно создавать:

- колодцы;
- дыры и внутренние пустоты;
- резкие перепады высот;
- выступы и неудобные карманы.

### `Coop`

Игрок помогает компьютеру прожить как можно дольше.

- стартовый счет: `0`;
- на каждом ходу предлагаются `4` фигуры;
- на выбор есть `4` секунды на старте;
- если время вышло, компьютер получает случайную фигуру из текущего набора;
- за такой пропуск по текущей логике игры снимается `25` очков;
- за успешно поставленную фигуру команда получает очки, равные размеру фигуры;
- за каждую очищенную линию команда получает еще `15` очков;
- партия заканчивается, когда следующую фигуру уже нельзя поставить.

С ростом счета увеличивается сложность:

- базовый диапазон фигур: от `4` до `5` клеток;
- каждые `500` очков время на выбор уменьшается на `300` мс;
- после `500` очков в наборе могут появляться 6-клеточные фигуры, но не больше `2` за один ход;
- после `1500` очков в одном наборе может быть до `3` фигур размером `6` клеток.

## Поле и ход

- размер поля: `7 x 16`;
- игра показывает `4` варианта на выбор;
- после выбора фигуры алгоритм перебирает все уникальные повороты и все допустимые позиции;
- для каждой постановки симулируется падение, очистка линий и оценка итогового состояния поля;
- затем выбирается лучший вариант.

Если фигура не помещается ни в одном повороте и ни в одной позиции, игра завершает раунд.

## Генерация фигур

Игра не хранит заранее фиксированный список фигур. Вместо этого она генерирует все связные полимино нужного размера:

- строит все формы из `N` клеток;
- приводит их к каноническому виду;
- убирает дубликаты;
- сохраняет только уникальные повороты;
- случайно собирает из пула набор из `4` разных фигур.

За счет этого партии не ограничиваются одними тетромино: набор быстро становится шире и менее предсказуемым.

## Специальная фигура: бомба

Помимо обычных полимино в пул добавляется `бомба`.

- бомба занимает `1` клетку;
- после падения она очищает квадрат `3 x 3` вокруг точки приземления;
- алгоритм старается использовать ее там, где взрыв уберет самые опасные высокие и плотные участки поля.

Бомба доступна в обоих режимах и встроена в общий набор предлагаемых фигур.

## Как компьютер выбирает ход

Алгоритм не использует нейросети, машинное обучение или внешние API. Все решения детерминированы и основаны на переборе.

При оценке позиции учитываются:

- количество дыр;
- глубина дыр;
- суммарная высота колонок;
- максимальная высота;
- перепады рельефа;
- переходы заполнено/пусто по строкам и столбцам;
- глубина колодцев;
- опасность верхних рядов;
- бонус за очищенные линии.

Поверх локальной оценки есть короткий lookahead:

- из текущего пула берется случайная выборка до `10` фигур;
- для каждой считается лучший следующий ход;
- в итоговую оценку добавляется поправка на худший сценарий.

Это помогает алгоритму не только переживать текущий ход, но и не загонять поле в тупик сразу после него.

## Технически

Проект полностью собран в одном файле: [`index.html`](./index.html).

Внутри находятся:

- HTML-разметка;
- CSS-интерфейс;
- игровая логика на чистом JavaScript;
- Web Audio-эффекты;
- генерация полимино и алгоритм оценки поля.

Сборщик, фреймворк и внешние зависимости не используются.

## Запуск

Достаточно открыть `index.html` в браузере. Отдельный сервер не нужен.
